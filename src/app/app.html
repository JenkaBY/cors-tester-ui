About

This application helps test HTTP endpoints and diagnose CORS (Cross-Origin Resource Sharing) issues by allowing you to send customized HTTP requests with specific headers and examine the responses in detail.

How to Use

  1. URL and Method: Enter the target URL and select the HTTP method (GET, POST, etc.).
  2. Headers: The app includes common CORS-related headers by default:
    • Content-Type: application/json - for proper JSON handling
    • Origin: http://localhost:4200 - to test CORS policies
    Add more headers as needed, one per line in the format "Header-Name: Value".
  3. Request Body: For POST/PUT/PATCH methods, enter your JSON request body.
  4. Send Request: Click the "Send Request" button to make the call.
  5. Response: Examine the:
    • HTTP status code and message
    • Response headers (check for CORS-related headers)
    • Response body

Common CORS Headers

  • Access-Control-Allow-Origin: Check which origins are allowed
  • Access-Control-Allow-Methods: Available HTTP methods
  • Access-Control-Allow-Headers: Permitted request headers
